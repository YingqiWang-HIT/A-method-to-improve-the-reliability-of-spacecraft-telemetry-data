# HSTGAT-CD Reference Implementation

This repository provides a PyTorch-based HSTGAT-CD reference implementation for recovering irregular spacecraft telemetry data. This implementation is based on a journal paper still under review. The paper describes three main components: a **VLSTM-SDE** temporal recovery module with continuous-time dynamics and stochastic perturbation, a **hierarchical graph attention** module using local/global/cross-subsystem graphs, and a **soft-hard parameter sharing** optimization strategy. The manuscript also reports a 6:2:2 split, Adam with learning rate 1e-4, 1000 epochs, early stopping patience 80, and 8 attention heads in HiGAT. fileciteturn2file0L10-L15 fileciteturn2file0L22-L25 fileciteturn1file1L11-L18

## Important note

This is a **faithful engineering refactoring**, not the author's official code release. Further details of the paper will be disclosed after its publication. Several implementation details in the manuscript are not fully specified, including exact MIC thresholds, some dimensional choices, and certain training-stage details. Wherever the paper is ambiguous, this repo uses **clearly documented defaults** so that the code can run end-to-end.

## Features

- Continuous-time telemetry recovery with a VLSTM-SDE-style recurrent cell
- Time-varying modulation signal `A(t), ω(t), φ(t)` generated by an MLP
- Brownian perturbation term for stochastic hidden dynamics
- Hierarchical graph construction:
  - local graph within subsystem
  - global graph between subsystems
  - cross-subsystem graph between linked subsystems
- Multi-head graph attention implemented in plain PyTorch
- Soft-sharing regularization within subsystems
- Training / evaluation / prediction scripts
- Synthetic irregular telemetry generator for immediate testing
- CSV-based custom dataset support

## Quick start with synthetic data

Generate a demo dataset:

```bash
python scripts/make_synthetic_data.py --output_dir demo_data
```

Train:

```bash
python scripts/train.py \
  --config src/hstgat_cd/configs/default.yaml \
  --data_path demo_data/telemetry.csv \
  --layout_path demo_data/subsystem_layout.json \
  --output_dir demo_run
```

Evaluate:

```bash
python scripts/evaluate.py \
  --config src/hstgat_cd/configs/default.yaml \
  --data_path demo_data/telemetry.csv \
  --layout_path demo_data/subsystem_layout.json \
  --checkpoint demo_run/best_model.pt
```

Predict and save recovered sequences:

```bash
python scripts/predict.py \
  --config src/hstgat_cd/configs/default.yaml \
  --data_path demo_data/telemetry.csv \
  --layout_path demo_data/subsystem_layout.json \
  --checkpoint demo_run/best_model.pt \
  --output_file demo_run/predictions.csv
```

## Input data format

Your CSV should contain:

- one time column: `timestamp`
- one column per telemetry channel
- missing values can be represented as empty cells or `NaN`

Example:

```csv
timestamp,sensor_1,sensor_2,sensor_3
0,0.12,0.51,
1,0.15,0.49,0.91
2,,0.48,0.88
```

## Subsystem layout JSON

Example:

```json
{
  "subsystems": {
    "attitude": ["sensor_1", "sensor_2", "sensor_3"],
    "thermal": ["sensor_4", "sensor_5"],
    "power": ["sensor_6", "sensor_7", "sensor_8"]
  }
}
```

This is used to build the hierarchical graph structure described in the paper. The manuscript explicitly states that telemetry is divided by subsystem, local graphs are built inside each subsystem, global graphs are built between subsystem feature descriptors, and cross-subsystem graphs are then created only for linked subsystem pairs. fileciteturn2file0L16-L23

## Paper-to-code mapping

### 1. VLSTM-SDE

The paper describes continuous forget/input/output gates, a time-varying modulation signal, and Brownian perturbation inside continuous-time state updates. fileciteturn2file0L12-L15 fileciteturn2file0L13-L14

In this repo:

- `VLSTMSDECell` implements gate computation
- `mod_mlp` outputs `A(t), ω(t), φ(t)`
- Euler-Maruyama style updates approximate the Ito-integral form
- `dt` is taken from timestamps and can vary per step

### 2. HiGAT

The paper describes a local graph `GA`, a global subsystem graph `GB`, and a cross-subsystem graph `GC`, followed by multi-head graph attention. fileciteturn2file0L16-L23

In this repo:

- `graph_builder.py` builds the three graph levels
- `graph_attention.py` implements multi-head masked graph attention
- `HSTGATCD` fuses temporal and graph modules end-to-end

### 3. Soft-hard parameter sharing

The paper uses soft-sharing regularization among temporal models within a subsystem and hard-sharing for the graph module. fileciteturn2file0L22-L23

In this repo:

- temporal soft sharing is implemented as subsystem-wise parameter similarity loss
- spatial module is a shared graph attention stack used across all nodes

## Defaults tied to the manuscript

The config uses the paper's reported training choices where possible:

- Adam optimizer
- learning rate = `1e-4`
- batch size = `32`
- early stopping patience = `80`
- graph attention heads = `8`
- normalization before train/val/test split
- 6:2:2 split
- synthetic masking rates from `{0.03, 0.05, 0.08, 0.10}`

These settings are directly stated in the manuscript. fileciteturn1file1L11-L18

## Reproducibility tips

- Start with synthetic data to verify the environment.
- Then replace `telemetry.csv` and `subsystem_layout.json` with your own dataset.
- If you have exact subsystem priors from your paper experiments, use them in the JSON layout.
- If you know the precise MIC thresholds used in your experiments, update them in the YAML config.
